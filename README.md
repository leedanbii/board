## Spring Boot 게시판 프로젝트

## 📝 프로젝트 소개
- 게시판 웹 애플리케이션 구현
- 회원가입, 로그인/로그아웃, 게시글 작성/조회/수정/삭제, 댓글 작성 기능 포함
- 인증 및 권한 기반 접근 제어 적용

---

# 🎯 기능 목록

## 1. 회원가입

- 사용자가 **아이디, 비밀번호, 이름**을 입력하면 회원가입을 한다.
- 아이디는 **5~20자**이며 **영어/숫자**로 이루어진 **영어로 시작**하는 문자여야 한다.
- 비밀번호는 **8~30자**이며 **영어/숫자/특수문자**가 모두 포함되어야 한다.
- 이름은 최대 **5글자**로 제한된다.
- 아이디는 다른 사용자와 **중복**될 수 없다.

**예외 상황**

- 입력한 아이디가 이미 존재하는 경우
- 이름이 5글자를 넘는 경우
- 아이디가 5~20자가 아닌 경우
- 아이디가 영어로 시작하지 않는 경우
- 아이디가 영어/숫자 외의 다른 문자를 포함하는 경우
- 비밀번호가 8~30자가 아닌 경우
- 비밀번호가 영어/숫자/특수문자 모두를 포함하지 않는 경우
- 비밀번호가 영어/숫자/특수문자 외의 다른 문자를 포함하는 경우

---

## 2. 로그인

- 사용자가 **아이디, 비밀번호**로 로그인을 한다.

**예외 상황**

- 아이디와 비밀번호가 일치하지 않을 경우
- 아이디가 존재하지 않는 경우

---

## 3. 게시글 작성

- 로그인 한 사용자가 **제목, 내용**을 입력한다.
- 제목은 **30자**, 내용은 **1000자**로 제한된다.

**예외 상황**

- 로그인 하지 않은 사용자가 게시글 작성을 시도하는 경우
- 제목이 30자를 초과하는 경우
- 내용이 1000자를 초과하는 경우

---

## 4. 게시글 목록 / 조회

- 사용자가 작성한 **게시글 목록**(제목/작성자/작성일)을 보여준다.
- 게시글 **상세 조회**(글 제목/내용/작성자/작성일)가 가능하다.

**예외 상황**

- 상세 조회하려는 게시글이 이미 삭제된 경우

---

## 5. 게시글 수정 / 삭제

- 게시글 상세 조회 시 **작성자**는 **게시글 수정/삭제**가 가능하다.

**예외 상황**

- 로그인 하지 않은 사용자가 게시글 수정/삭제를 시도하는 경우
- 작성자가 아닌 사용자가 게시글 수정/삭제를 시도하는 경우

---

## 5. 댓글 작성 / 삭제

- 로그인 한 사용자는 모든 게시글에 댓글을 작성할 수 있다.
- 댓글 내용은 1자 이상 300자 이하여야 한다.
- 댓글 삭제는 댓글을 작성한 작성자만 삭제할 수 있다.

**예외 상황**

- 로그인 하지 않은 사용자가 댓글 작성/삭제를 시도하는 경우
- 작성자가 아닌 사용자가 댓글 삭제를 시도하는 경우
- 댓글 내용을 작성하지 않거나 공백만을 작성하는 경우
- 댓글이 300자를 초과하는 경우

# 🛠️ 기술 스택 및 구조

이번 프로젝트에서는 Spring 기반의 웹 애플리케이션을 구현하며, 아래와 같은 기술과 구조를 사용했습니다.

## 기술 스택

- **Spring Boot**: 웹 애플리케이션 개발 및 의존성 관리
- **Spring Security**: 인증(Authentication) 및 권한(Authorization) 처리, 비밀번호 암호화
- **JPA (Hibernate)**: 객체-관계 매핑(ORM)을 통한 데이터베이스 연동
- **MySQL**: 관계형 데이터베이스
- **Thymeleaf**: 서버 사이드 HTML 렌더링
- **Maven**: 빌드 및 의존성 관리


## 🏗️ 구조 및 아키텍처

### 레이어 구조
- **Controller**: 요청 처리 및 View 반환
- **Service**: 비즈니스 로직 처리 및 권한 검증
- **Repository**: JPA를 통한 DB 연동
- **Domain**: Board, User, Comment 등의 핵심 엔티티
- **Security**: Spring Security 기반 인증/권한 관리


### 특징

- **Spring Security**를 활용하여 로그인, 로그아웃, 접근 권한 관리를 구현
- **JPA**를 통해 User, Board 등의 엔티티를 DB 테이블과 매핑
- **MySQL 데이터베이스**와 연동하며, `spring.jpa.hibernate.ddl-auto=update`로 스키마 자동 업데이트
- 게시글 작성, 조회, 수정, 삭제 시 인증 및 권한 검증을 서비스 계층에서 처리

---

## 🔒 인증 및 권한 관리
- 로그인 사용자는 게시글 작성 가능
- 게시글/댓글 수정 및 삭제는 작성자만 가능
- Thymeleaf `#authentication.name`으로 현재 사용자 ID 확인 후 UI 제어

---

## 💾 데이터베이스 구조
- **User 테이블**: id, name, userId, password
- **Board 테이블**: id, title, content, writer_id, createdAt
- **Comment 테이블**: id, content, board_id, commenter_id, createdAt
- **관계**
    - User 1:N Board
    - Board 1:N Comment
    - User 1:N Comment

---

## 🚀 실행 방법
1. MySQL 실행 및 데이터베이스 생성
2. `application.properties`에서 DB 정보 확인
3. 프로젝트 빌드 및 실행
4. 브라우저에서 `http://localhost:8080/` 접속
