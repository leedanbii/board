# 🎯 기능 목록

## 1. 회원가입

- 사용자가 **아이디, 비밀번호, 이름**을 입력하면 회원가입을 한다.
- 이름은 최대 **5글자**로 제한된다.
- 아이디는 다른 사용자와 **중복**될 수 없다.

**예외 상황**

- 입력한 아이디가 이미 존재하는 경우
- 이름이 5글자를 넘는 경우

---

## 2. 로그인

- 사용자가 **아이디, 비밀번호**로 로그인을 한다.

**예외 상황**

- 아이디와 비밀번호가 일치하지 않을 경우
- 아이디가 존재하지 않는 경우

---

## 3. 게시글 작성

- 로그인 한 사용자가 **제목, 내용**을 입력한다.
- 제목은 **30자**, 내용은 **1000자**로 제한된다.

**예외 상황**

- 로그인 하지 않은 사용자가 게시글 작성을 시도하는 경우
- 제목이 30자를 초과하는 경우
- 내용이 1000자를 초과하는 경우

---

## 4. 게시글 목록 / 조회

- 사용자가 작성한 **게시글 목록**(제목/작성자/작성일)을 보여준다.
- 게시글 **상세 조회**(글 제목/내용/작성자/작성일)가 가능하다.

**예외 상황**

- 상세 조회하려는 게시글이 삭제된 경우

---

## 5. 게시글 수정 / 삭제

- 게시글 상세 조회 시 **작성자**는 **게시글 수정/삭제**가 가능하다.

**예외 상황**

- 로그인 하지 않은 사용자가 게시글 수정/삭제를 시도하는 경우
- 작성자가 아닌 사용자가 게시글 수정/삭제를 시도하는 경우

---

# 🛠️ 기술 스택 및 구조

이번 프로젝트에서는 Spring 기반의 웹 애플리케이션을 구현하며, 아래와 같은 기술과 구조를 사용했습니다.

## 기술 스택

- **Spring Boot**: 웹 애플리케이션 개발 및 의존성 관리
- **Spring Security**: 인증(Authentication) 및 권한(Authorization) 처리, 비밀번호 암호화
- **JPA (Hibernate)**: 객체-관계 매핑(ORM)을 통한 데이터베이스 연동
- **MySQL**: 관계형 데이터베이스
- **Thymeleaf**: 서버 사이드 HTML 렌더링
- **Maven**: 빌드 및 의존성 관리

## 구조

---

### 특징

- **Spring Security**를 활용하여 로그인, 로그아웃, 접근 권한 관리를 구현
- **JPA**를 통해 User, Board 등의 엔티티를 DB 테이블과 매핑
- **MySQL 데이터베이스**와 연동하며, `spring.jpa.hibernate.ddl-auto=update`로 스키마 자동 업데이트
- 게시글 작성, 조회, 수정, 삭제 시 인증 및 권한 검증을 서비스 계층에서 처리

---

# 📌 기타 사항

- RESTful하게 URL 구성 (`/boards`, `/boards/{id}/update`, `/boards/{id}/delete`)
- Spring Boot 3.x, Spring Security 6.x 기반 구현
